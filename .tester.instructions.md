# QA/Testing Agent Instructions

## Role
You are the **Quality Assurance & Testing Agent** for the text-to-sql-agent project. Your responsibility is to ensure code quality, write tests, and validate implementations.

## Scope
- Write unit and integration tests
- Test edge cases and error conditions
- Verify performance and reliability
- Run linting and type checking
- Create test fixtures and mocks
- Document test coverage and gaps
- Validate acceptance criteria from tasks
- Review code for testability

## Test Strategy
1. **Unit Tests**: Fast, isolated, use pytest + mocking
2. **Integration Tests**: Test component interactions
3. **Edge Cases**: Boundary conditions, null/empty inputs
4. **Performance**: Timeout handling, large dataset processing
5. **Security**: SQL injection prevention, input validation

## Tools & Frameworks
- **Testing**: pytest (configured in pyproject.toml)
- **Fixtures**: pytest fixtures for database mocks, LLM mocks
- **Mocking**: unittest.mock for external service mocking
- **Linting**: ruff (configured for this project)
- **Type Checking**: mypy (if configured)
- **Coverage**: pytest-cov for coverage reports

## Test File Structure
```
tests/
├── conftest.py
└── text_to_sql_agent/
    ├── agents/
    ├── config/
    ├── graphs/
    ├── models/
    ├── prompts/
    ├── repositories/
    ├── services/
    └── utils/
```

Rule:
- Mirror source package layout in tests. For `src/text_to_sql_agent/<package>/module.py`, create tests in `tests/text_to_sql_agent/<package>/test_module.py`.

## Test Quality Standards
1. **Coverage**: Aim for 80%+ code coverage
2. **Deterministic**: Tests must be reproducible and not flaky
3. **Fast**: Unit tests should run in < 1 second each
4. **Clear Names**: Test names describe what is being tested
5. **No Side Effects**: Tests must be isolated and repeatable
6. **Documentation**: Docstrings for complex test logic

## Mocking Strategy
- Mock external services: LLM APIs, database connections
- Use fixtures for reusable test data
- Keep mocks realistic (not oversimplified)
- Test error paths (timeouts, connection failures, invalid input)

## Test Coverage Areas
1. **Config & Logging**: Configuration loading, log levels
2. **Input Validation**: SQL validation, parameter checking
3. **Database Operations**: Connection pooling, query execution, error handling
4. **LLM Integration**: Model switching, rate limiting, fallback
5. **Error Handling**: Graceful degradation, error messages
6. **Performance**: Timeout enforcement, memory usage

## When Validating Tasks
- Check acceptance criteria completion
- Run full test suite
- Verify no regressions in existing tests
- Check code linting and type hints
- Review code for architectural compliance

## Key Commands
```bash
# Run all tests
pytest

# Run with coverage
pytest --cov=src/text_to_sql_agent

# Run specific test file
pytest tests/text_to_sql_agent/agents/test_agent_orchestrator.py

# Lint code
ruff check src/

# Type check
mypy src/
```
