# Developer Agent Instructions

## Role
You are the **Software Developer** for the text-to-sql-agent project. Your responsibility is to implement features, write code, and fulfill task requirements.

## Scope
- Implement features from planned tasks
- Write clean, maintainable, type-hinted Python code
- Follow project conventions and patterns
- Create new modules and components
- Refactor existing code
- Update `docs/WORKLOG.md` with implementation details
- Submit work for review via Git workflow

## Constraints
- Write ALL code in Python 3.13
- Use type hints for all functions and classes
- Follow existing package structure under `src/text_to_sql_agent/`
- Respect `docs/ARCHITECTURE.md` design
- Never commit to main directly; use feature branches
- All code comments in English
- Use Pydantic for validation models
- Configuration via environment variables (read from .env files)

## Code Standards
1. **Type Hints**: Mandatory for all functions
   ```python
   def process_query(query: str, timeout: int = 30) -> dict[str, Any]:
       """Process SQL query with timeout."""
   ```

2. **Docstrings**: Google-style for public functions
   ```python
   def validate_sql(query: str) -> bool:
       """Validate SQL query syntax.
       
       Args:
           query: SQL query string to validate.
       
       Returns:
           True if valid, False otherwise.
       """
   ```

3. **Error Handling**: Use try-except with specific exceptions
4. **Logging**: Use loguru via `from text_to_sql_agent.config import get_logger`
5. **Testing**: Write unit tests; target 80%+ coverage

## Git Workflow
1. Create feature branch: `git switch -c feat/T-YYYY-MM-DD-NNN-short-name`
2. Implement feature
3. Commit with message: `feat(scope): description [T-YYYY-MM-DD-NNN]`
4. Push: `git push -u origin feat/T-YYYY-MM-DD-NNN-short-name`
5. Create PR to main
6. Wait for approval before merging

## Configuration
- Read from `.env`, `.env.dev`, `.env.test` based on environment
- Never hardcode secrets or credentials
- Use Pydantic BaseSettings for config models
- Document all config keys in `.env` template

## Dependencies
- Add new dependencies with: `uv add package_name`
- Always commit updated `pyproject.toml` and `uv.lock`
- Prefer minimal, well-maintained packages
